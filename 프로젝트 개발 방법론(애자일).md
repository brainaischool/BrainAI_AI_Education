### **자주 발생하는 문제 & 해결**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
애자일 운영 시 FAQ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Q1: "Sprint 목표를 못 이뤘어요!"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
A: 괜찮습니다! 애자일의 핵심은 실패 허용

해결 방법:
1. Retrospective에서 원인 분석
2. 다음 Sprint에 남은 Task 이월
3. 목표를 더 작게 조정

예시:
Sprint 2 목표: 500장 수집
실제: 300장만 수집
→ Sprint 3에서 200장 추가 수집!


Q2: "팀원이 안 참여해요!"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
A: Daily Standup에서 즉시 파악

해결 방법:
1. 역할 재분배
2. 1:1 면담
3. 흥미있는 Task 배정

예시:
"코딩 싫어해요" → 데이터 수집 담당
"발표 싫어해요" → 문서 작성 담당


Q3: "시간이 부족해요!"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
A: 우선순위 재조정

해결 방법:
1. Must Have vs Nice to Have 구분
2. 핵심만 먼저 완성
3. 추가 기능은 나중에

예시:
Must: 3바퀴 완주
Nice: 장애물 회피, 신호등 인식
→ Must만 우선!


Q4: "너무 쉬워요! 빨리 끝났어요!"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
A: 좋은 신호! 추가 도전 부여

해결 방법:
1. Stretch Goal 설정
2. 다른 팀 돕기
3. 심화 과제 제공

예시:
기본: 3바퀴 완주
Stretch: 5바퀴 + 더 빠르게!


Q5: "Retrospective가 형식적이에요"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
A: 질문 방식 바꾸기

재미있는 질문:
😊 "이번 주 MVP(최고 선수)는?"
😢 "다시는 안 할 것?"
💡 "다음 주 한 가지만 바꾼다면?"

활동 추가:
- 포스트잇으로 익명 의견
- 타임라인 그리기
- 감정 곡선 그래프
```
