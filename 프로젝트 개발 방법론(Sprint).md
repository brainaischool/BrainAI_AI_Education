## 📅 Sprint의 구조

### **Sprint는 이렇게 생겼어요**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Sprint 2 예시 (1주 = 5일)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📅 월요일 (Sprint 시작!)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔵 Sprint Planning (15분)
   "이번 주 목표: 데이터 300장 수집!"
   
   할 일 목록:
   □ 카메라 연결
   □ 주행 연습
   □ 데이터 수집 (100장)
   □ 데이터 확인


📅 화요일~목요일 (실행!)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
매일 아침:
🔵 Daily Standup (5분)
   "어제: 100장 모았어요"
   "오늘: 100장 더 모을 거예요"
   "문제: 배터리 부족"

그 다음:
💻 실제 작업!
   화요일: 100장 수집 ✅
   수요일: 100장 수집 ✅
   목요일: 100장 수집 ✅


📅 금요일 (Sprint 끝!)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔵 Sprint Review (15분)
   "우리 300장 모았어요!" 🎉
   [실제로 보여주기]

🔵 Retrospective (10분)
   😊 "뭐가 좋았어?"
   😢 "뭐가 안 좋았어?"
   💡 "뭘 개선할까?"

🔵 다음 Sprint 준비
   "다음 주 목표: 데이터 정제!"


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Sprint 2 완료! ✅
Sprint 3 시작! →
```

```

너무 짧으면: 계획 시간만 걸림
너무 길면: 지루함


BrainAI Car 프로젝트:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1 Sprint = 1주 (2차시 수업)

왜 1주?
✓ 고등학생 집중력 고려
✓ 매주 성취감
✓ 빠른 피드백
✓ 7주면 프로젝트 완성!


시간표 예시:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

또는
```

## 🎯 Sprint의 핵심 규칙

### **4가지 원칙**

```
1. 기간은 고정! ⏰
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❌ "아직 안 끝났으니까 3일 연장!"
✅ "1주 끝! 남은 건 다음 Sprint에!"

→ 시간 관리 능력 키우기


2. 목표는 하나! 🎯
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❌ "이번 주에 데이터도 모으고,
    모델도 훈련하고, 발표 준비도..."
    
✅ "이번 주는 데이터 수집만!"

→ 집중력 높이기


3. 중간에 변경 금지! 🚫
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❌ "Sprint 중간에 목표 바꾸기"

✅ "다음 Sprint에 새 목표 추가"

→ 계획의 중요성 배우기


4. 결과물 필수! 📦
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❌ "열심히 했어요!" (증거 없음)

✅ "300장 모았어요!" (파일 있음)

→ 실제 결과로 증명
```

## 💬 Sprint 용어 정리

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Sprint 관련 용어 사전
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Sprint
   = 짧은 기간 (1-2주)
   예: "Sprint 2가 시작됐어요!"


2. Sprint Goal
   = 이번 Sprint의 목표
   예: "데이터 300장 수집"


3. Sprint Planning
   = Sprint 시작 회의
   예: "이번 주 뭐 할까?"


4. Daily Standup
   = 매일 아침 짧은 회의
   예: "어제/오늘/문제" 공유


5. Sprint Review
   = Sprint 끝 시연
   예: "300장 모았어요! (보여주기)"


6. Retrospective
   = 회고, 돌아보기
   예: "뭐가 좋았고, 뭘 개선할까?"


7. Sprint Backlog
   = Sprint 동안 할 일 목록
   예: 
   □ 카메라 연결
   □ 데이터 수집
   □ 데이터 확인


8. Burndown Chart
   = 진행률 그래프
   예: 
   Day 1: 남은 일 10개
   Day 3: 남은 일 5개
   Day 5: 남은 일 0개! ✅
```
