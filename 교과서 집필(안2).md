```
#### **3.3 우리 프로젝트의 4가지 설정 파일**

```
📄 1. hardware.yaml - 하드웨어 설정서
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

# 시리얼 통신 (마이크로비트)
serial:
  baudrate: 115200         # 통신 속도
  
# 카메라
camera:
  source: 0                # 0번 카메라 사용
  width: 640
  height: 480
  
# 모터
motor:
  default_speed: 600       # 🎯 여기를 바꾸면 속도 변경!
  forward_range: [181, 1023]  # 속도 범위
  
# 서보 (핸들)
servo:
  center: 90               # 직진 각도
  left: 135                # 왼쪽 끝
  right: 45                # 오른쪽 끝


🤔 이 파일은 언제 열어볼까?
→ 차량 속도를 바꾸고 싶을 때
→ 카메라가 이상할 때
→ 핸들 범위를 조정하고 싶을 때


─────────────────────────────────────

📄 2. data_collection.yaml - 데이터 수집 설정서
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

collection:
  save_dir: "data_laneD1"  # 🎯 저장 위치
  save_fps: 12             # 초당 12장 저장
  
image:
  format: "jpg"            # JPG 형식
  size: [224, 224]         # 크기 조절

recording:
  button: "R1"             # R1 버튼으로 녹화
  delete_button: "L1"      # L1 버튼으로 삭제
  delete_count: 10         # 10장씩 삭제


🤔 이 파일은 언제 열어볼까?
→ 더 많은 사진을 찍고 싶을 때 (fps 증가)
→ 저장 위치를 바꾸고 싶을 때


─────────────────────────────────────

📄 3. training.yaml - AI 훈련 설정서
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

model:
  name: "mobilenet_v2"     # 사용할 AI 모델
  input_size: [224, 224, 3]
  
hyperparameters:
  batch_size: 32           # 한 번에 32장씩
  epochs: 50               # 🎯 50번 반복 학습
  learning_rate: 0.001     # 학습 속도
  
paths:
  data_dir: "data_laneD1"
  model_save: "models/lane_model.h5"


🤔 이 파일은 언제 열어볼까?
→ 학습 시간을 줄이고 싶을 때 (epochs 감소)
→ 성능을 높이고 싶을 때 (epochs 증가)
→ 다른 데이터로 훈련할 때 (data_dir 변경)


─────────────────────────────────────

📄 4. deployment.yaml - 자율주행 설정서
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

model:
  path: "models/final/lane_model.xml"
  use_openvino: true       # 최적화된 모델 사용
  
control:
  base_speed: 450          # 🎯 자율주행 속도
  max_speed: 600
  
  steering:
    sensitivity: 1.0       # 핸들 민감도
    smoothing: 0.3         # 부드러움 정도

safety:
  emergency_key: "q"       # Q키로 긴급 정지


🤔 이 파일은 언제 열어볼까?
→ 자율주행이 너무 빠를 때 (base_speed 감소)
→ 핸들이 너무 많이 꺾일 때 (sensitivity 감소)
→ 주행이 떨릴 때 (smoothing 증가)
```

#### **4.1 이름 하나로 모든 게 달라진다!**

```
❌ 나쁜 예시:

data/
├── 1.jpg
├── 2.jpg
├── a.json
├── b.json
├── 최종.jpg
├── 최최종.jpg
└── 진짜최종.jpg

"어떤 게 뭐죠? 😱"


✅ 좋은 예시:

data/raw/laneD1/images/
├── 0001.jpg
├── 0002.jpg
├── 0003.jpg
...

data/raw/laneD1/annotations/
├── 0001.json
├── 0002.json
├── 0003.json
...

"0001.jpg랑 0001.json은 짝꿍이구나! 😊"
```

#### **4.2 우리 프로젝트 네이밍 규칙**

```
📸 이미지 파일 이름
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

형식: 0001.jpg, 0002.jpg, ...

규칙:
✓ 4자리 숫자 (0001~9999)
✓ 순서대로 증가
✓ 앞에 0을 채워서 정렬 편하게

예시:
0001.jpg  ← 첫 번째 사진
0002.jpg  ← 두 번째 사진
...
0350.jpg  ← 350번째 사진

형식: 0001.json, 0002.json, ...

규칙:
