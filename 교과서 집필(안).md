## 🎯 교과서의 핵심 차별화 포인트

### **1. SW 공학 → 구현 순서** (기획 우선!)

```
기존 교재:
코딩부터 → "일단 만들고 보자"

본 교재:
기획·설계 → "왜 이렇게 설계했는가?"
```

### **2. 실무 중심 산출물**

| 단계 | 산출물 | 실무 활용도 |
|-----|--------|------------|
| 기획 | 요구사항 명세서 | ⭐⭐⭐⭐⭐ |
| 설계 | 시스템 아키텍처 문서 | ⭐⭐⭐⭐⭐ |
| 설계 | 디렉터리 구조 설계서 | ⭐⭐⭐⭐ |
| 설계 | API 명세서 | ⭐⭐⭐⭐⭐ |
| 구현 | 코드 + 주석 | ⭐⭐⭐⭐ |
| 테스트 | 테스트 결과 보고서 | ⭐⭐⭐⭐ |

### **3. 사고의 흐름**

```
❌ 일반 교재:
"이 코드를 실행하세요" → 왜? (X)

✅ 본 교재:
"왜 configs/ 폴더가 필요한가?" 
→ 설정과 코드 분리 원칙
→ 유지보수성 향상
→ 그래서 만든다
```

Chapter 3. 디렉터리 구조 설계
─────────────────────────────────────
3.1 설계 원칙

    1) Separation of Concerns (관심사 분리)
       - configs/: 설정과 코드 분리
       - utils/: 재사용 가능한 모듈
       - notebooks/: 학습/실습 분리
       
    2) Single Responsibility (단일 책임)
       - 각 모듈은 하나의 책임만
       - car_controller.py ≠ data_collector.py
       
    3) DRY (Don't Repeat Yourself)
       - constants.py: 공통 상수 한 곳에
       - config_loader.py: 설정 로드 재사용

Chapter 7. AI 모델 설계
─────────────────────────────────────
7.1 문제 정의

    [Task]
    회귀 문제 (Regression)
    
    [Input]
    - 이미지: (224, 224, 3) RGB
    
    [Output]
    - 조향값: float (-1.0 ~ 1.0)
    
    [손실 함수]
    - MSE (Mean Squared Error)
    
    [평가 지표]
    - MAE (Mean Absolute Error)
    - 실제 주행 성공률

7.2 모델 아키텍처 설계

    [선택: MobileNetV2]
    
    Input (224, 224, 3)
         ↓
    MobileNetV2 (ImageNet 사전학습)
         ↓
    GlobalAveragePooling2D
         ↓
    Dense(128, activation='relu')
         ↓
    Dropout(0.3)
         ↓
    Dense(1, activation='tanh')  # -1 ~ 1
         ↓
    Output (steering angle)
    
    [설계 결정 근거]
    - MobileNetV2: 경량화 (임베디드)
    - 전이학습: 적은 데이터로 학습
    - Tanh 활성화: -1~1 범위 보장

7.3 훈련 전략 설계

    [하이퍼파라미터]
    - Batch Size: 32
    - Epochs: 50
    - Learning Rate: 0.001
    - Optimizer: Adam
    
    [데이터 증강]
    - 좌우 반전 (+ 조향값 반전)
    - 밝기 조절 (0.8 ~ 1.2)
    - 약간의 회전 (±5도)
    
    [콜백 설계]
    - EarlyStopping: val_loss 10 에포크
    - ModelCheckpoint: 최고 성능 저장
    - ReduceLROnPlateau: 학습률 감소

7.4 배포 전략 설계

    [최적화]
    Keras Model (.h5)
         ↓
    OpenVINO Toolkit
         ↓
    Optimized Model (.xml + .bin)
    
    [성능 목표]
    - 추론 속도: 30 FPS 이상
    - 정확도: MAE < 0.1
    - 메모리: < 100MB
